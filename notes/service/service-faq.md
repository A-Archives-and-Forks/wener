# Service FAQ

## Workflow vs Serverless

可以互为补充

- Workflow
  - 侧重业务、流程、编排、自动化
  - 业务逻辑实现
  - 特点：流程定义、状态管理、可视化和监控
- Serverless
  - 侧重代码、runtime、ingress、webhook
  - 事件驱动应用
  - 可能运行 job, cron task, event handler, webhook, delay task, long running server
  - 特点：按需部署、按时长计费、无状态、无运维、可视化和监控
