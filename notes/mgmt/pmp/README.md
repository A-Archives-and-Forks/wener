# PMP

| abbr. | stand for                            | meaning                  |
| ----- | ------------------------------------ | ------------------------ | ------------ |
| BRD   | Business Requirements Document       | 商业需求文档             |
| MRD   | Market Requirements Document         | 市场需求文档             |
| PMBOK | Project Management Body of Knowledge | 项目管理知识体系         |
| PMI   | Project Management Institute         | 项目管理协会             |
| PMO   | Project Management Office            | 项目管理办公室           |
| PMP   | Project Management Professional      | 项目管理专业人士         |
| PRD   | Product Requirements Document        | 产品需求文档             |
| SOW   | Statement of Work                    | 工作说明书               |
| WBS   |                                      | Work Breakdown Structure | 工作分解结构 |

- https://pmichina.org/
- 考试时间
  - 3、6、9、12 月
  - 4 小时
- 考试内容
  - 180 道选择题
  - 含单选和多选题
- 考试费用 3900
- PDU - Professional Development Units - 专业发展单元
  - 1 小时即累积 1 个 PDU
  - 三年为续证周期
  - 一般 60 个 PDU
- PDH

---

- 临时性、独特性
- 时间、范围、成本
- 质量、风险、资源
- 启动、规划、执行、监控、收尾
- 建筑工程、信息技术、市场营销、医疗保健、金融服务、活动策划

| 文档类别       | 核心文档                     | 主要目的                     | 主要读者           | 主要撰写人                 |
| -------------- | ---------------------------- | ---------------------------- | ------------------ | -------------------------- |
| 商业与产品定义 | BRD, MRD, PRD                | 定义为什么做、做什么、怎么做 | 管理层、产品、研发 | 产品经理、业务分析师       |
| 项目授权与启动 | Project Charter              | 正式启动项目，明确目标和授权 | 全体干系人         | 项目发起人、项目经理       |
| 项目规划蓝图   | Project Management Plan, WBS | 详细规划范围、时间、成本     | 项目团队           | 项目经理                   |
| 进度与风险跟踪 | Status Report, Risk Register | 监控项目健康状况             | 项目经理、干系人   | 项目经理                   |
| 变更管理       | Change Request Form          | 规范化地处理项目变更         | 变更控制委员会     | 提出变更的干系人、项目经理 |
| 知识传承       | Lessons Learned Report       | 总结经验教训                 | 公司、未来项目团队 | 项目经理、全体团队成员     |

| 英文             | 中文                |

| 文档名称 (Document Name)            | 阶段 / 类别   | 核心目的 (Core Purpose)            | 主要撰写人                 | 主要读者                 | 关键要素 (Key Elements)                      |
| ----------------------------------- | ------------- | ---------------------------------- | -------------------------- | ------------------------ | -------------------------------------------- |
| 项目启动 (Initiation)               |               |                                    |                            |                          |
| 项目章程 (Project Charter)          | 启动阶段      | 正式批准项目，授予项目经理权力     | 项目发起人、项目经理       | 全体干系人               | 项目目标、高阶需求、预算概览、项目经理任命   |
| 商业论证 (Business Case)            | 启动阶段      | 阐明项目的商业价值和可行性         | 业务分析师、产品经理       | 公司管理层、决策者       | 成本效益分析、市场机会、ROI预测、风险评估    |
| 项目规划 (Planning)                 |               |                                    |                            |                          |
| 需求与范围                          |               |                                    |                            |                          |
| PRD (产品需求文档)                  | 规划阶段      | 详细定义产品的功能与特性           | 产品经理                   | 研发、设计、测试团队     | 用户故事/功能列表、交互逻辑、非功能性需求    |
| WBS (工作分解结构)                  | 规划阶段      | 将项目范围分解为可管理的工作包     | 项目经理、团队核心成员     | 项目团队                 | 可交付成果的层级分解、工作包、WBS词典        |
| 范围说明书 (Scope Statement)        | 规划阶段      | 清晰界定项目的边界（做与不做）     | 项目经理                   | 项目团队、关键干系人     | 项目范围描述、验收标准、项目除外责任         |
| 人与职责                            |               |                                    |                            |                          |
| RACI 矩阵 (RACI Matrix)             | 规划阶段      | 明确各项任务的角色与职责           | 项目经理                   | 项目团队、关键干系人     | 任务列表、角色分配 (R/A/C/I)                 |
| 资源管理计划 (Resource Plan)        | 规划阶段      | 规划如何获取、管理和使用资源       | 项目经理                   | 项目团队、职能经理       | 资源需求、角色职责、资源日历、获取策略       |
| 干系人参与计划 (Stakeholder Plan)   | 规划阶段      | 制定与干系人有效互动的策略         | 项目经理                   | 项目团队、沟通团队       | 干系人列表、影响/利益分析、参与策略          |
| 质量与采购                          |               |                                    |                            |                          |
| 质量管理计划 (Quality Plan)         | 规划阶段      | 定义项目的质量标准与保证方法       | 项目经理、质量经理         | 项目团队、测试团队       | 质量标准、质量控制活动、验收流程、工具       |
| SOW (工作说明书)                    | 规划阶段      | 详细定义外包给供应商的工作         | 项目经理、采购部门         | 供应商、项目团队         | 工作范围、交付物、时间表、付款条件、验收标准 |
| 执行与监控 (Execution & Monitoring) |               |                                    |                            |                          |
| 问题日志 (Issue Log)                | 执行与监控    | 记录并跟踪项目中已发生的问题       | 全体成员发现、项目经理管理 | 项目团队                 | 问题描述、优先级、负责人、状态、解决方案     |
| 决策日志 (Decision Log)             | 执行与监控    | 记录项目过程中的关键决策           | 项目经理                   | 项目团队、关键干系人     | 决策内容、决策依据、决策日期、决策者         |
| 状态报告 (Status Report)            | 执行与监控    | 定期汇报项目进展、风险和健康状况   | 项目经理                   | 项目干系人、管理层       | 进度更新、成本状况、风险预警、里程碑状态     |
| 变更请求单 (Change Request)         | 执行与监控    | 规范化地管理对项目基准的变更       | 任何干系人、项目经理       | 变更控制委员会 (CCB)     | 变更描述、影响分析、理由、审批意见           |
| 项目收尾 (Closing)                  |               |                                    |                            |                          |
| 复盘/总结报告 (Lessons Learned)     | 收尾阶段      | 总结经验教训以改进未来项目         | 项目经理、全体团队         | 公司管理层、未来项目团队 | 成功经验、失败原因分析、具体改进建议         |
| 敏捷核心产物 (Agile Artifacts)      |               |                                    |                            |                          |
| 产品待办列表 (Product Backlog)      | 敏捷 (全周期) | 动态管理所有产品需求               | 产品负责人                 | 敏捷团队、干系人         | 用户故事、功能项、Bug修复、优先级排序        |
| Sprint 待办列表 (Sprint Backlog)    | 敏捷 (执行期) | 开发团队在一个 Sprint 中承诺的工作 | 开发团队                   | 开发团队                 | Sprint 目标、选定的用户故事、任务分解        |
| 完成的定义 (Definition of Done)     | 敏捷 (全周期) | 确保团队对“完成”有统一的质量标准   | 开发团队                   | 产品负责人、开发团队     | 代码审查、自动化测试、文档更新等检查项       |
